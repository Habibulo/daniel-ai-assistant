Review checklist and PR standards. Check for: correctness, edge cases, security vulnerabilities, performance regressions, naming clarity, test coverage. PR description must include what changed, why, and how to test. Small focused PRs over large omnibus PRs. Review within 24 hours. Constructive feedback with suggested alternatives.
